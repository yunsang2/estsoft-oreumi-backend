DOM 이벤트

 브라우저를 사용하면서 어떤 상호작용이 있으면, 브라우저는 이벤트를 발생시킨다
 1. 사용자의 동작에 의해 발생하는 상호작용 - 클릭, 특정요소에 마우스를 가져가거나 특정 키 누름
 2. 브라우저의 상태변화에 의한 상호작용 - 로딩의 끝, 화면 스크롤, 크기 변경

 JavaScript 에서 특정이벤트에 어떤일을 할지 설정 할 수 있다


    이벤트 관련 용어
     이벤트 이름, 타입
     이벤트를 구분하기 위한 식별자


     이벤트 속성
      객체의 속성으로 객체에 지벤트가 발생하면 브라우저는 객체의 이벤트 속성에 연결된 이벤트 헨들러(함수)를 실행한다


     이벤트 핸들러 or 이벤트 리스너(더 많이 사용)
     이벤트가 발생했을때 실행할 기능(함수)

     DOM 이벤트 모델에 연결 가능한 3가지

        인라인 이벤트 모델
         HTML 속성으로 이벤트 속성을 설정하는 형태

         속성 값으로 실행할 코드를 지정하는데, HTML 마크업에 코드가 추가된형태이기에 적절하지 않음
         (쓰지 않는다)


        고전 이벤트 모델
         DOM 객체의 이벤트 속성에 이벤트 핸들러를 대입하는 형태

         인라인 이벤트 모델에 비해 보다 유연하게 이벤트를 처리할 수 있다
         
         고전 이벤트 모델은 중첩해서 등록시 하나의 이벤트만 적용되기에 마지막에 작성된 이벤트만 적용된다
         그래서 중첩해서 사용할 수 없다
         

        표준 이벤트 모델
         addEventListener메서드로 이벤트 핸들러를 연결하는 형태

         한 이벤트에 여러 이벤트를 연결할 수 있다 이를 이벤트 핸들러 체인이라 한다

         이벤트 캡처링을 지원한다

         IE8 과 이전 버전에서는 지원하지 않는다


    이벤트 객체
     이벤트와 관련된 정보와 기능을 담고있는 객체
    
     이벤트가 발생할때마다 브라우저는 이벤트 객체를 생성하고, 이벤트 핸들러를 실행하면서 이벤트 객체를 인자로 넣어준다
    
     이벤트 핸들러에서는 매개 변수에 전달된 이벤트 객체를 통해 이벤트의 상태를 파악하고, 필요한 경우 조작할 수 있다
    

    기본 이벤트
     일부 HTML 요소는 따로 지정하지 않아도 기본적으로 설정된 기능을 가진다
    
     <a>, <input>, <form> 요소등 이 있다
     

    기본 이벤트 제거
     이런 기본 이벤트가 동작하지 않아야 하는 경우, 객테에 이벤트 핸들러를 연결하고
     이벤트 핸들러에서 기본 이벤트를 제거해 실행되지 않도록 한다
     매개변수.preventDefault(); - 매개변수에 지정된 이벤트 제거


    이벤트 전달
     부모, 자식요소에 적용된 이벤트를 자식, 부모 요소에 전달해 같은 이벤트를 발생
     기본 이벤트와 달리 한가지 방법밖에 없다
     clock, mousemove, change, submit 이벤트등 이있다


    이벤트 전달 제거
     매개변수.stopPropagation(); - 매개변수에 지정된 이벤트 전달 제거

    window 객체의 load 이벤트
     load 이벤트는 windiw 객체를 로드하는 게 끝나면 발생한다
     브라우저가 HTML 문서의 내용을 읽고 DOM 트리 완성 문서에 연결된 스타일시트, javascript 파일, 이미지등 외부 자원을 모두 불러온 순간이 window 객체의 로드가 완료되는 시점이다
     모든 작업이 끝난후 실행


    document 객체의 DOMContentLoaded 이벤트
     브라우저가 HTML 문서의 내용을 읽고 DOM 트리를 완성한 순간 실행
     외부자원을 불러오는걸 기다리지않기에 이미지의 크기를 제대로 구하지 못할 수도 있다
     DOMContentLoaded 이벤트에 해당하는 이벤트 속성은 없고 addEventListener 메서드로 연결해야한다
     Explorer 11 이하 버전은 지원하지 않는다


    이벤트 객체의 속성
        bubbles – 이벤트가 버블링 되는지 여부
        cancelable – preventDefault()로 기본 동작을 취소할 수 있는지 여부
        currentTarget – 현재 이벤트 리스너가 등록된 요소
        defaultPrevented – 기본 동작이 이미 취소되었는지 표시
        eventPhase – 이벤트가 현재 어떤 전파 단계에 있는지 (캡처 / 타깃 / 버블링)
        target – 실제 이벤트가 처음 발생한 요소
        timeStamp – 이벤트가 발생한 시간(밀리초)
        type – 이벤트의 종류(ex: "click", "mousemove")
        view – 이벤트가 발생한 환경(window 객체)


    이벤트 객체의 메서드
        preventDefault() – 해당 이벤트의 기본 동작을 취소함
         (예: 링크 이동, 폼 제출 등 기본 동작 막기)
        stopPropagation() – 이벤트가 부모 요소로 버블링되는 것을 중단함
        stopImmediatePropagation() – 동일 요소에 등록된 다른 이벤트 리스너도 실행되지 않도록 완전히 중단함
        composedPath() – 이벤트가 전파되는 경로(타깃 → 부모들)를 배열 형태로 반환


    DOM 마우스 이벤트
        click - 마우스 클릭
        dblclick - 마우스 더블 클릭
        mousedown - 마우스 누를때
        mouseup - 마우스 뗄 때
        mousemove - 마우스 움직일때
        mouseenter - 포인터가 HTML 요소 경계 외부에서 내부로 이동할 때
        mouseleave - 포인터가 HTML 요소 경계 내부에서 외부로 이동할 때
        mouseover - 포인터가 HTML 요소 경계 외부에서 내부로 이동할 때(이벤트 전달)
        mouseout - 포인터가 HTML 요소 경계 내부에서 외부로 이동할 때(이벤트 전달)


    MouseEvent 객체의 속성
        altKey, ctrlKey – Alt / Ctrl 키가 눌린 상태인지 여부
        metaKey, shiftKey – Meta(윈도우/Command) / Shift 키가 눌린 상태인지 여부
        button – 클릭한 마우스 버튼 번호(0: 왼쪽, 1: 휠, 2: 오른쪽)
        buttons – 현재 눌려 있는 모든 버튼 상태를 비트 값으로 표시
        relatedTarget – mouseover/mouseout 등에서 이전 또는 다음으로 이동한 요소
        movementX, movementY – 이전 이벤트와 비교한 마우스 이동 변화량(px)
        clientX, clientY – 브라우저 화면(Viewport) 기준의 마우스 좌표
        pageX, pageY – 문서 전체(Page) 기준의 마우스 좌표(스크롤 포함)
        screenX, screenY – 모니터 스크린 기준의 마우스 좌표


    DOM 키보드 이벤트
        keydown - 키보드의 키가 눌러질 때 발생
        keypress - 글자가 입력될 때 발생(한글일 경우 지원하지 않음)
        keyup - 키보드의 키가 떼어질 때 발생


    KeyboardEvent 객체의 속성
        altKey, ctrlKey – Alt / Ctrl 키가 눌린 상태인지 여부
        metaKey, shiftKey – Meta(윈도우/Command) / Shift 키가 눌린 상태인지 여부
        code – 키보드의 물리적 키 값 (예: KeyA, Digit1, ArrowUp)
        키보드 레이아웃과 무관하게 특정 키 자체를 구분할 때 사용
        key – 입력된 문자 값 또는 키 이름 (예: "a", "1", "Enter", "ArrowLeft")
        키보드 입력 상태(한/영, 대소문자 등)에 따라 달라질 수 있음
        location – 키의 위치 정보 (0: 일반, 1: 왼쪽, 2: 오른쪽, 3: 숫자패드)
        repeat – 키가 눌린 상태로 유지되어 이벤트가 반복 발생 중인지 여부

    
    윈도우 이벤트
        abort – 리소스 로딩이 중단되었을 때 발생
        error – 스크립트 오류나 리소스 로딩 오류가 발생했을 때
        hashchange – URL의 # 해시값이 변경될 때
        load – 문서와 모든 리소스(img, script 등)가 모두 로딩 완료되었을 때
        pageshow – 페이지가 표시될 때 (뒤로 가기 캐시에서도 발생)
        pagehide – 페이지가 숨겨질 때 (다른 페이지로 이동하거나 탭 이동 시)
        resize – 브라우저 창 크기가 변경될 때
        scroll – 문서가 스크롤될 때
        unload – 페이지가 언로드(떠날 때) 될 때 발생


    DOM 입력 양식 이벤트
        blur – 요소가 포커스를 잃었을 때
        change – 입력값이 변경된 후 포커스를 벗어났을 때
        focus – 요소가 포커스를 얻었을 때
        focusin – 포커스를 얻을 때(버블링 O)
        focusout – 포커스를 잃을 때(버블링 O)
        input – 사용자 입력이 발생할 때마다 즉시 실행
        reset – 폼이 초기화될 때
        select – 텍스트가 선택될 때
        submit – 폼 제출 시 발생
