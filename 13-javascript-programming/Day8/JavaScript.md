JavaScript 함수와 객체

    <템플릿 리터럴>
     표현식을 내장할 수 있는 문자열 리터럴을 말하며, 여러줄에 걸친 문자열을 만들수 있다
     (``)를 사용해 문자열을 작성하면 작성한 문자열중 개행도 포함하며 출력
     문자열 안에 ${} 를 사용해 해당 자리에 들어갈 값을 지정가능하다 (표현식, 값...)
    

    <함수>
     특정한 기능을 수행하는 프로그램의 코드를 모아놓은것
     한번 작성해 놓으면 나중에 필요할때 계속 불러 쓸수 있다
     프로그램 내에서 다루는 값만 다를 뿐만 같은 토드를 사용할수 있기에 미리 작성해놓는 편이다
     
    <함수의 선언>
     function 키워드로 선언하며 식별자를 지정한다

     function hello() {
        console.log('Hello World!');
     }

    <함수의 호출>
     프로그램이 실행되면서 function 키워드를 만나면 함수가 선언되러 있다는 것만 확인하고 지나친다
     함수 호출 표현식으로 함수를 호출해야 함수 본체에 작성한 문장들이 실행된다
     (함수 식별자 + 함수 호출 연산자) 로 호출
     
     hello();

    <함수의 자료형>
     함수는 자료형이 function인, 프로그램 코드의 집합 을 나타내는 값이다
     자료형이 function인 값은 함수 호출 연산자('()')로 호출할 수 있다

    <함수의 대입>
     함수는 자료형이 function인 값이다 따라서 변수에 대입할 수 있다
     함수를 변수에 대입하면 변수의 식별자로도 함수에 접근할 수 있다

     function hello() {
        console.log('Hello World!');
     }
     var world = hello;

     world(); === hello();

    <익명 함수>
    함수를 선언하면서 식별자를 지정하지 않은 함수
    함수를 변수에 대입하면 변수의 식별자로 함수에 접근할 수 있다  즉 변수의 식별자롸 함술 호출 연산자로 함수를 호출할 수 있다 
    이때 식별자를 생략할 수 있다

    <매개 변수>
     함수가 기정된 기능을 수행하는데 필요한 값을 전달받는 변수
     함수를 선언할때 함수 식별자 다음에 나오는 괄호 안에 쉼표(,)로 구분하여 나열
     매개변수는 함수를 호출하면 선언 되고 인자로 초기화 된다
     매개 변수는 지역변수이므로 함수 안에서만 접근할수있고 함수의 실행이 끝나면 사라진다
     함수 호출시 매개변수의 개수와 맞게 인자를 지정해야한다 그렇지 않으면 초기화하지 못한다

     function name(name) {
        console.log('name =', name);
     }

     name(sangee); === 'name = sangee'

    <지역 변수>
     함수 본체에서 var 키워드로 선언한 변수로, 선언한 함수 본테 내에서만 접근할 수 있다
     함수의 매개변수도 지역변수다

    return 문
     실행중인 함수를 종료하고 프로그램 실행 흐름은 함수 호출 표현식으로 되돌아간다
     return 뒤에 어떤 값을 지정하면 실행이 끝난후 되돌아가면서 값을 가지고 감
     반환 값이 없다면 return문은 생략가능


    var, let, const 키워드

        var 키워드
         var 키워드로 선언한 변수는 같은 식별자로 여러번 중복해서 선언할 수 있다
         var 키워드로 선언한 변수는 함수레벨스코프를 가진다
         var 키워드로 선언한 변수는 호이스팅으로 인해 변수를 선언하기 전에 접근할 수 있다

        let, const 키워드
         let 키워드와 const 키워드로 선언한 변수는 같은 식별자로 중복해서 선언할 수 없다
         let 키워드와 const 키워드로 선언한 변수는 블록레벨스코프를 가진다
         let 키워드로 선언한 변수는 재할당할 수 있지만 const 키워드로 선언한 변수는 상수이므로 초기화해야하고 재할당할 수 없다
         const 키워드로 선언후 재 선언을 한다면 let 키워드로 변경하는것이 좋다


    <객체>
     객체는 어떤 대상을 표현하는 여러 값들과 기능을 하나로 묶어서 다루는 자료형이다.
     객체는 객체 생성 리터럴({})로 생성하며, 속성들을 객체 리터럴 안에 나열한다
     속성은 키와 값의 쌍으로 이루어져 있으며, 콜론으로(:) 구분된다
     인덱스 연산자를 통해 해당 속성에 접근하려면 반드시 문자열 형태의 키를 줘야 접근가능하다
     객체의 속성에 접근하는 방법은 2가지로 나뉘며
     1. 식별자, 인덱스 연산자([]), 문자열 형태의 키
     2. 식별자, 속성 접근 연산자(.), 식별자 형태의 키

        {this}
         메서드 본체에서 사용하며, 메서드에 접근하고 있는 객체를 나타내는 키워드
        
        {for-in}
         객체의 속성에 차례대로 접근(하지만 거의 쓰지 않는다)

window 객체의 속성, 메서드

    window.innerHeight
     뷰포트의 높이
    window.outHeight
     윈도우 창의 높이
        
    window 객체의 타이머 관련 메서드
     객체에서 제공하는 타이머
     지정한 시간이 지난 다음, 지정한 기능을 수행하는 타이머
     지정한 시간 간격마다, 지정한 기능을 반복적으로 수행하는 타이머
    
    타이머를 등록하는 window객체의 메서드
    
        window.setTimeout 메서드
         지정한 시간이 지난 다음, 지정한 기능을 수행하는 타이머를 브라우저에 등록
            window.setTimeout(함수, 시간 (ms));

        window.setInterval 메서드
         지정한 시간 간격마다, 지정한 기능을 반복적으로 수행하는 타이머를 브라우저에 등록
            window.setInterval(함수, 시간 (ms));

        타이머가 등록되면 브라우저는 각 타이머에 ID 를 지정해주고 해당 ID로 각 타이머에 접근가능하다
 
        브라우저, 즉 window 객체에 등록된 타이머를 해제하는 메서드
         window.setTimeout 메서드로 지정된 타이머 해제
            window.clearTimeout(ID);
 
        window.setInterval 메서드로 지정된 타이머를 해제
            window.clearInterval(ID);
