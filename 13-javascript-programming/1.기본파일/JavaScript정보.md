    <자료형과 변수>

        {자료형}
         프로그램에서 다루는 값의 종류
         자료형에 따라 프로그램에서 값을 다루는 방법이 달라진다
            [string]      문자열
             ('', "")로 묶어서 표현 가능하지만 되도록 같은 따옴표를 사용
             (+)으로 문자열을 연결해 출력할수있다 하지만 숫자랑 문자열은 더할 수 없다
             피연산자들중 하나라도 문자열이면 다른 자료형을 문자열로 변환후 연결한다            
            [number]      숫자
             특별한 값
                Infinity    무한대(숫자를 0으로 나누면 연산결과가 무한이 나옴)
                NaN         숫자로 나타낼수 없는 값을 의미하는 키워드
            [undefined]   선언되지 않았거나 할당되지 않은 값
             변수로 선언하지 않은 식별자
             변수로 선언을 했지만 초기화하지 않은 변수
            [boolean]     불린(참, 거짓)
            [function]    함수
             일정한 기능을하는 역할을 가진 기계와 같다
             function 식별자() { 기능; } 라는 형태로 작성
             함수 호출 연산자'()'는 자료형이 함수인 형태뒤에만 붙는다
             함수 호출 표현식을 통해 실행된다
            [object]      객체


        {typeof} 연산자
         typeof 연산자로 값의 자료형을 확인할 수 있다
         typeof 연산자의 연산 결과는 문자열이다


        {변수}
         변하는 값, 값을 저장하기 위한 저장공간에 붙이는 식별자를 말한다
         프로그램에서 일시적으로 필요한 값을 일시적으로 저장할 목적으로 사용
         var 키워드로 선언하며, 대입 연산자(=)로 값을 대입한다.
         변수를 선언할때 값을 대입하는것을 변수의 초기화 라고 한다


        {상수}
         변하지 않는 값, "대입"을 통해 상수를 변수에 "복사"할 수 있다


        {배열}
         유형이 같은 여러 값들을 저장하기 위한 저장공간들, 즉 변수들의 집합
         배열은 객체(object)의 한 종류이다 각 저장공간은 원소 라고 하며 0부터 1씩 분서대로 증가한다
         배열의 원소들의 개수를 "배열의 크기(length) 라고 한다" 
         마지막 인덱스는 (length - 1)이다
         배열은 배열 리터럴([])로 생성하며, 변수에 대입해서 접근한다
         또한 다른 자료형을 추가하거나 대입할 수 있다(c++이랑 다름)
         
    <연산자>

        {표현식}
         값을 만들어 내는 코드
         연산자와 피연산자로 이루어진다
         표현식은 항상 결과 값을 가진다, 즉 연산자는 항상 연산 결과를 가진다
            [대입 연산자]
             변수와 같은 저장공간에 값을 저장하는 연산자
             우변에 있는 값을 좌변에 있는 변수와 같은 저장공간에 대입한다

            [산술 연산자]
             평소 사용하는 계산 술식들( +, -, *, /, ** )

            [복합 대입 연산자]
             산술 연산자와 대입 연산자가 복합된 연산자이며 대입 연산자의 일종이다
             대입 연산자와 같이 좌변의 값은 변수 등 대입 가능한 값이 와야한다

            [단항 연산자]
             피연산자가 하나인 연산자
                +num            변수의 값을 양수로 변환 
                -num            변수의 값을 음수로 변환 
                ++num, num++    변수의 값에 1을 더함, 식이 연산된후 계산
                --num, num--    변수의 값에 1을 뺌  , 식이 연산된후 계산

            [이항 연산자]
             피연산자가 두 개인 연산자

            [삼항 연산자]
             피연산자가 세 개인 연산자

            [비교 연산자]
             이항 연산자이며 양쪽에 있는 피연산자의 값을 비교
             '<', '>' 등등 크기 비교는 알고있는것과 일치한다
                ==          서로의 값이 같은지 비교
                !=          서로의 값이 다른지 비교
                ===         서로의 자료형이 같은지 비교 같으면 값을 비교
                !==         서로의 자료형이 다른지 비교 같으면 값을 비교

            [논리 연산자]
             피연산자를 참 또는 거짓으로 다루고, 연산 결과도 참 또는 거짓이 되는 연산자
                !       피연산자가 참이면 거짓, 거짓이면 참(단항 연산자)
                &&      피연산자가 모두 참이면 참, 아니면 거짓(이항 연산자)
                ||      피연산자 중에서 하나라도 참이면 참, 모두 거짓이면 거짓(이항 연산자)

            [조건 연산자]
             조건에 따라 다른 표현식을 연산 결과로 취하는 연산자
             피연산자가 3개다, 참이면 2번째 피연산자가 결과가 되고, 거짓이면 3번째 피연산자가 결과가된다
                (참 or  거짓) ? 참일때 연산결과 : 거짓일때 연산결과
            
    <제어문>

        {복합문}
         중괄호{}로 묶어 놓은 것, 블록 이라고도 한다
         문장 하나가 들어가야할 자리에 여러 문장들을 넣어야 하는 경우 사용한다
        
        {조건문}
         만약에 ~ 하면 ~ 하겠다 라는 식의 구문을 조건문이라한다
            [if 구문]
             조건이 참인지 거짓인지에 따라 참이면 절차를 수행
             거짓이면 실행되지 않고 넘어감
                
                if (조건식) {
                    참일시 실행;
                }
            
            [if else 구문]
             조건이 참인지 거짓인지에 따라 참이면 절차를 수행
             거짓이면 else 이후에 절차를 수행
                
                if (조건식) {
                    참일시 실행;
                }
                else {
                    거짓일시 실행;
                }

            [if else if 구문]
             조건문에 조건문을 중첩한 경우

                if (조건식) {
                    참일시 아래 조건식을 실행
                    if (조건식) {
                        참일시 실행;
                    }
                    else {
                        거짓일시 실행;
                    }
                }
                else {
                    거짓일시 실행;
                }

        {반복문}
         
            [while 구문]
             가장 기본적인 반복문
             주어진 조건식이 참인 동안 계속해서 실행 거짓이면 반복문 종료

                while (조건식)
                    참인동안 실행;

            [do while 구문]
             코드 블록이 먼저 실행하고, 나중에 조건을 검사하는 반복문
             코드 블록을 최소한 한 번은 실행해야하는 경우 사용

                do
                    참일시 실행(하지만 먼저 실행)
                while (조건식);

            [for 구문]
             while 구문은 조건에 비중을 두지만 for 구문은 반복횟수에 비중을 둔다

                for (초기화; 조건문; 실행후 실행되는 식)
                    참일시 실행;

    <객체>
     객체는 어떤 대상을 표현하는 여러 값들과 기능을 하나로 묶어서 다루는 자료형이다.
     객체는 객체 생성 리터럴({})로 생성하며, 속성들을 객체 리터럴 안에 나열한다
     속성은 키와 값의 쌍으로 이루어져 있으며, 콜론으로(:) 구분된다
     객체의 속성에 접근하는 방법은 2가지로 나뉘며
     1. 식별자, 인덱스 연산자([]), 문자열 형태의 키
     2. 식별자, 속성 접근 연산자(.), 식별자 형태의 키

        {this}
         메서드 본체에서 사용하며, 메서드에 접근하고 있는 객체를 나타내는 키워드
        
        {for-in}
         객체의 속성에 차례대로 접근(하지만 거의 쓰지 않는다)
        


window 객체의 타이머 관련 메서드
 객체에서 제공하는 타이머
  지정한 시간이 지난 다음, 지정한 기능을 수행하는 타이머
  지정한 시간 간격마다, 지정한 기능을 반복적으로 수행하는 타이머

    타이머를 등록하는 window객테의 메서드
        window.setTimeout 메서드
         지정한 시간이 지난 다음, 지정한 기능을 수행하는 타이머를 브라우저에 등록
            window.setTimeout(함수, 시간 (ms));
        window.setInterval 메서드
         지정한 시간 간격마다, 지정한 기능을 반복적으로 수행하는 타이머를 브라우저에 등록
            window.setInterval(함수, 시간 (ms));

        타이머가 등록되면 브라우저는 각 타이머에 ID 를 지정해주고 해당 ID로 각 타이머에 접근가능하다
 
        브라우저, 즉 window 객체에 등록된 타이머를 해제하는 메서드
         window.setTimeout 메서드로 지정된 타이머 해제
            window.clearTimeout(ID);
 
        window.setInterval 메서드로 지정된 타이머를 해제
            window.clearInterval(ID);

 1. 사용자의 동작에 의해 발생하는 상호작용 - 클릭, 특정요소에 마우스를 가져가거나 특정 키 누름
 2. 브라우저의 상태변화에 의한 상호작용 - 로딩의 끝, 화면 스크롤, 크기 변경

             인라인 이벤트 모델에 비해 보다 유연하게 이벤트를 처리할수 있다


    이벤트 객체
     이벤트와 관련된 정보와 기능을 담고있는 객체
     이벤트가 발생할때마다 브라우저는 이벤트 객체를 생성하고, 이벤트 핸들러를 실행하면서 이벤트 객체를 인자로 넣어준다
     이벤트 핸들러에서는 매개 변수에 전달된 이벤트 객체를 통해 이벤트의 상태를 파악하고, 필요한 경우 조작할 수 있다
    
        기본 이벤트
         일부 HTML 요소는 따로 지정하지 않아도 기본적으로 설정된 기능을 가진다
         이런 기본 이벤트가 동작하지 않아야 하는 경우, 객테에 이벤트 핸들러를 연결하고
         이벤트 핸들러에서 기본 이벤트를 제거해 실행되지 않도록 한다
         <a>, <input>, <form> 요소등 이 있다
            매개변수.preventDefaulr(); - 매개변수에 지정된 이벤트 제거

        이벤트 전달
         부모, 자식요소에 적용된 이벤트를 자식, 부모 요소에 전달해 같은 이벤트를 발생
         기본 이벤트와 달리 한가지 방법밖에 없다
         clock, mousemove, change, submit 이벤트등 이있다
            매개변수.stopPropagation(); - 매개변수에 지정된 이벤트 전달 제거

        표준 이벤트 모델
            객체.addEventListener(eventName, handlener);
            객체.
